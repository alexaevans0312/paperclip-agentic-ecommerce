# Demo Frame — Paperclip Agentic Ecommerce

## Objective

Convince the judges that Paperclip is a real agentic business operating layer for ecommerce: useful, viable, and presentable.

## Audience

- Nous Research judges evaluating Hermes usage.
- Stripe judges looking for real earn/spend/provision workflows.
- NVIDIA judges looking for meaningful model orchestration and reasoning use.
- Ecommerce operators/investors who understand that CAC, compliance, and execution discipline matter.

## Narrative

Paperclip is not another AI marketing assistant. It is a founder-controlled operating layer that turns ecommerce intent into action:

1. Understand the business.
2. Build the offer.
3. Check the economics.
4. Clean the claims.
5. Queue the spend/provisioning plan.
6. Review with Nemotron.
7. Hand the founder an execution packet.

## Visual language

- Founder chaos → clean operating rail.
- Warm Squish brand colours, but restrained enough for an infrastructure/business audience.
- Cards for each specialist agent.
- Ledger-style UI for Stripe approval gates.
- Scorecard-style UI for economics and compliance.

## Proof points to show

- Real Squish source context.
- BRD/PRD/TRD in repo.
- Skills catalog and workflows.
- Stripe approval ledger.
- Nemotron review packet.
- Founder next actions.

## Tone

Confident, operational, investor-legible.

Avoid:

- “AI magic.”
- generic “growth hacking.”
- pretending live payments happened if they did not.
- skincare medical claims.
