# Demo Plan — Paperclip Agentic Ecommerce

## Demo goal

Show that Paperclip is an agentic ecommerce operating layer for a real DTC brand, not a novelty chatbot.

Target runtime: 1-3 minutes.

## Core message

Paperclip helps one founder run the early operating surface of an ecommerce brand: campaign strategy, offer economics, compliance, Stripe-governed spend/provisioning, and execution handoff.

## Demo structure

### 0:00-0:15 — Set the business context

Show:

- Squish is a real pre-launch pimple patch brand.
- The founder needs to launch ecommerce without a large team.
- The challenge: earn, spend, and run operations safely.

Suggested narration:

> This is Paperclip, an agentic ecommerce operating layer for Squish. Squish is launching premium visible hydrocolloid pimple patches, and Paperclip turns founder goals into operational workflows a lean business can actually use.

### 0:15-0:40 — Founder brief

Show the brief:

> Launch Squish's first pimple patch offer with a bundle-led Shopify funnel and creator seeding plan while keeping CAC under $18 and all claims cosmetic-safe.

Paperclip decomposes into:

- strategy,
- ecommerce economics,
- compliance,
- Stripe ops,
- Nemotron review.

### 0:40-1:15 — Operating packet output

Show generated outputs:

- offer structure,
- funnel map,
- creator acquisition plan,
- unit economics check,
- compliance-safe copy.

Key line:

> The system rejects marketing that cannot support the unit economics or creates skincare claim risk.

### 1:15-1:45 — Stripe skills and approval ledger

Show Stripe action plan:

- `stripe-projects`: provision a development database or dashboard service for campaign tracking.
- `stripe-link-cli`: approval-gated operational purchase.
- `mpp-agent`: pay-per-call enrichment/API usage with cap.

Show ledger:

- pending approval,
- estimated cost,
- spend cap,
- environment,
- verification command.

Key line:

> Paperclip can spend or provision, but it does not silently spend. Payment actions become audited business events.

### 1:45-2:15 — Nemotron 3 Ultra orchestration layer

Show Nemotron review packet/result:

- viability check,
- missing dependency check,
- payment/process risk review,
- final recommendation.

Key line:

> Nemotron 3 Ultra acts as the high-capacity operations reviewer, stress-testing the plan before Hermes executes scoped actions.

### 2:15-2:45 — Why it matters

Show final founder next-actions list.

Suggested narration:

> This is the operating model behind Squish: one founder, specialised agents, Stripe-governed spend, and investor-legible ecommerce execution.
